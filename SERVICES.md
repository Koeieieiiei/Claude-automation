## 1. 🟣 Stripe — รับชำระเงิน

### ทำอะไร
ระบบรับเงินจากลูกค้า ตอนลูกค้ากดซื้อ ebook เงินจะวิ่งผ่าน Stripe แล้วเข้าบัญชีธนาคารของเรา ปัจจุบันตั้งให้รับผ่าน **PromptPay (สแกน QR) อย่างเดียว** (ตัดช่องบัตรเครดิตออกแล้ว)

### สถานะปัจจุบัน
- ✅ **โหมด Live (รับเงินจริง)** — ใช้คีย์ `sk_live_` / `pk_live_`
- ✅ ผูกบัญชีธนาคารสำหรับเบิกเงินแล้ว
- ✅ เปิด PromptPay ในโหมด Live แล้ว

### ราคา
- **ไม่มีค่ารายเดือน** จ่ายเป็นค่าธรรมเนียมต่อรายการเท่านั้น
- **PromptPay ≈ 2.2% ต่อรายการ** (เช่น ขาย 299 บาท เสียค่าธรรมเนียม ~7 บาท)

### ต้องจ่าย subscription ตอนไหน
- ❌ **ไม่มี subscription** — Stripe ไม่คิดรายเดือน คิดแค่เปอร์เซ็นต์ต่อยอดขาย ใช้ฟรีตลอดไม่ว่าจะขายมากหรือน้อย

### รายละเอียดที่ควรรู้
- 💰 เงินที่รับมาจะถูกพักไว้ช่วงแรก (~7–14 วันสำหรับบัญชีใหม่) ก่อนโอนเข้าบัญชีธนาคารจริง
- 🧪 **เวลาทดสอบระบบ ให้สลับไปใช้ "โหมดทดสอบ (Test mode)"** จะได้ใช้ QR จำลองฟรี ไม่เสียเงินจริง (มุมขวาบนของแดชบอร์ด)
- ⚠️ โหมด Test กับ Live ตั้งค่าแยกกันคนละชุด (คีย์, PromptPay, webhook ต้องตั้งทั้งสองฝั่ง)
- 🔑 อย่าเปิดเผยคีย์ `sk_live_` เด็ดขาด — ถ้าหลุดต้องรีบสร้างใหม่ทันที

## 2. 🟢 Supabase — ฐานข้อมูล + เก็บไฟล์

### ทำอะไร
ทำ 2 หน้าที่:
1. **เก็บไฟล์ ebook PDF** (ใน storage bucket ชื่อ `ebooks`)
2. **เก็บข้อมูลคำสั่งซื้อ / ลูกค้า** (ออเดอร์, อีเมล, สถานะการจ่ายเงิน)

### สถานะปัจจุบัน
- ✅ ใช้แพ็ก **Free** อยู่

### ราคา
- **Free:** ฟรี — ฐานข้อมูล 500MB, เก็บไฟล์ 1GB
  - ⚠️ ข้อจำกัด: ถ้าไม่มีคนใช้งานติดต่อกัน 7 วัน **โปรเจกต์จะถูกพัก** (กดปลุกกลับมาได้)
- **Pro:** **$25/เดือน (~900 บาท)** — ไม่โดนพัก, เก็บไฟล์ 100GB, สำรองข้อมูล (backup) รายวัน

### ควรอัปเกรด Pro ตอนไหน (เจอข้อใดข้อหนึ่ง)
1. 🔴 โปรเจกต์เริ่ม**โดนพักบ่อย** จนลูกค้ากดซื้อไม่ได้
2. 🔴 ไฟล์ ebook รวมกัน**ใกล้ 1GB**
3. 🔴 มี**ลูกค้าจริงสม่ำเสมอ** (ขายได้ทุกวัน) → อยากได้ความเสถียร + backup กันข้อมูลหาย

### รายละเอียดที่ควรรู้
- ช่วงเปิดตัวเล็กๆ แพ็กฟรีพอใช้ แต่ระวังเรื่อง "โดนพักหลังไม่มีคนใช้ 7 วัน"
- ถ้าเริ่มขายจริงจัง แนะนำขึ้น Pro เพื่อความอุ่นใจ (มี backup เผื่อข้อมูลหาย)

## 3. 🔵 Resend — ส่งอีเมลไฟล์ให้ลูกค้า

### ทำอะไร
หลังลูกค้าจ่ายเงินสำเร็จ ระบบจะส่งอีเมลพร้อมลิงก์ดาวน์โหลด ebook ให้ลูกค้าอัตโนมัติ โดยส่งจากอีเมล `noreply@tpat3mock.com`

### สถานะปัจจุบัน
- ✅ ใช้แพ็ก **Free**
- ✅ **Verify โดเมนเรียบร้อยแล้ว** (DKIM + SPF + DMARC ครบใน Namecheap)
- ✅ ส่งถึงได้ทุกค่าย (Gmail, Hotmail/Outlook, Yahoo ฯลฯ)

### ราคา
- **Free:** ฟรี — **3,000 อีเมล/เดือน** (100 ฉบับ/วัน)
- **Pro:** **$20/เดือน (~720 บาท)** — 50,000 อีเมล/เดือน

### ควรอัปเกรด Pro ตอนไหน
- 🔴 เมื่อขายเกิน **~3,000 ออเดอร์/เดือน** หรือส่งอีเมลเกินโควต้าฟรี (ยังอีกไกล)

### รายละเอียดที่ควรรู้
- 📨 อีเมลส่งถึงทุกค่าย แต่ **Hotmail/Outlook ชอบกรองเข้า Junk/Spam** สำหรับผู้ส่งหน้าใหม่
  - วิธีแก้: เมื่อเจออีเมลใน Junk ให้กด **"Not junk"** → ครั้งหน้าจะเข้า Inbox
  - ยิ่งส่งอีเมลออกไปเรื่อยๆ ชื่อเสียงผู้ส่งจะดีขึ้น เข้า Inbox ง่ายขึ้นเอง
- 💡 ควรมีข้อความบนเว็บเตือนลูกค้าว่า *"หากไม่พบอีเมล กรุณาเช็กกล่อง Junk/Spam"*

## 4. ⚫ Vercel — โฮสต์เว็บไซต์

### ทำอะไร
เป็นที่ฝากเว็บไซต์ให้คนทั่วไปเข้าถึงได้จริงผ่าน `tpat3mock.com` (Vercel เป็นเจ้าของเดียวกับ Next.js ที่เว็บนี้ใช้ เลยเข้ากันได้ดีที่สุด)

### สถานะปัจจุบัน
- ⚠️ ใช้แพ็ก **Hobby (ฟรี)** อยู่ — แต่ตอนนี้รับเงินจริงแล้ว

### ราคา
- **Hobby:** ฟรี — **แต่กติกาห้ามใช้เชิงพาณิชย์** (เว็บขายของเข้าข่ายนี้)
- **Pro:** **$20/เดือน (~720 บาท)**

### ควรอัปเกรด Pro ตอนไหน
- 🔴 **ควรอัปเกรดเร็วๆ นี้** — เพราะตอนนี้เว็บรับเงินจริงแล้ว = ใช้เชิงพาณิชย์ ซึ่งแพ็ก Hobby (ฟรี) ไม่อนุญาตตามกติกา มีความเสี่ยงโดน Vercel ระงับบัญชี (เว็บล่มกลางคันตอนมีลูกค้า)
- ➡️ **นี่คือบริการที่ควรจ่ายเป็นอันดับแรก**

### รายละเอียดที่ควรรู้
- 🚀 วิธี deploy โค้ดใหม่ขึ้นเว็บ: ใช้ Vercel CLI ในโฟลเดอร์โปรเจกต์
  ```
  npx vercel --prod --yes
  ```
  (ถ้า token หมดอายุ ต้อง `npx vercel login` ก่อน — ทำครั้งเดียวจบ)
- 📝 โปรเจกต์นี้ยัง **ไม่ได้เชื่อมกับ Git** จึงต้อง deploy ผ่าน CLI ในเครื่องทุกครั้ง
  - 💡 ถ้าในอนาคตเชื่อม GitHub ไว้ จะ deploy อัตโนมัติทุกครั้งที่แก้โค้ด สะดวกขึ้นมาก
